ⓘ Assumptions & Engineering Logic

▶

Section Structure: Every section contains exactly 2 rows. An 8-panel section has 4 modules per row; a 10-panel section has 5 modules per row.

Post Count: Each section uses 4 posts (2 per row, at each end of the beam). This is a simplified assumption; real designs may add intermediate posts for longer spans.

Beam / Rail Count: Each section uses 2 beams (1 per row). Each beam spans the full width of its row.

Clamp Logic: Mid clamps are placed between adjacent modules in a row. End clamps are placed at the outer edges. For N panels per row: mid clamps = (N−1)×2 per section, end clamps = 4 per section (2 per row × 2 ends).

Splices / Joiners: 2 per section — assumes one splice per beam for field-joined rail segments.

Bonding / Grounding: 1 bonding clip per panel, 1 grounding lug per section. Simplified for estimating.

Footprint Estimation: In portrait orientation, panel width is the horizontal span per module and panel length is the slope span. In landscape, these swap. Section width = panels per row × module horizontal span. Section depth = 2 × slope span + row spacing. Total array width = section width (uniform). Total array length = sections × section depth + (sections−1) × section spacing.

Inverter Sizing: Base logic is 1 inverter per system up to the inverter's max DC input. If total DC exceeds max input, count increases by ratio (rounded up). This is a simplified placeholder.

Waste Factor: Applied to structural hardware categories (clamps, splices, wire mgmt, fasteners, brackets, bonding). Not applied to modules, inverters, or optimizers.

Extra Hardware Allowance: When enabled, adds 3% extra clamps and 5% extra wire management clips on top of base quantities (before waste factor).

Miscellaneous Adders: When enabled, adds one "Miscellaneous Hardware Allowance" line item per project at a fixed quantity of 1 kit.

</> Developer Data Mapping Guide

▶

All embedded data lives at the top of the <script> section in clearly labeled constants. When migrating from spreadsheets, replace or extend these arrays/objects:

const PANELS = [...] — Panel catalog. Each entry: id, manufacturer, model, wattage, lengthIn, widthIn, cellType, notes.

const INVERTERS = [...] — Inverter catalog. Each entry: id, manufacturer, model, acKw, dcMaxKw, notes.

const FOUNDATIONS = [...] — Foundation/post types with id, name, description.

const STRUCTURAL_MEMBERS = [...] — Beam/rail types with id, name, description.

const SECTION_RULES = {...} — Per-section BOM rules keyed by section type ("8" or "10"). Each defines posts, beams, midClamps, endClamps, splices, bondingClips, groundLugs, wireMgmt, hardwareKits.

const BOM_ITEMS = {...} — Master item reference keyed by item key. Each has name, category, unit, sku, wasteEligible flag.

const APP_DEFAULTS = {...} — Default form values for reset.

const CALC_CONSTANTS = {...} — Thresholds (inverter sizing ratio), extra hardware percentages, misc adder quantities.

To add new section types (e.g., 12-panel sections), add a new key to SECTION_RULES and a new <option> to the Section Type dropdown.
